Phần 1: Trả lời Lý thuyết

1. Sự khác biệt giữa position: absolute và position: relative

Khác biệt: position: relative giữ phần tử ở nguyên vị trí mặc định trong luồng tài liệu, các thẻ khác không bị ảnh hưởng. Trong khi đó, position: absolute sẽ "nhấc" phần tử đó ra khỏi luồng mặc định, nó sẽ định vị dựa trên phần tử cha gần nhất có cài đặt position (khác static).

Ví dụ Responsive: Khi làm biểu tượng "số lượng" màu đỏ nằm đè lên "giỏ hàng", ta đặt giỏ hàng là relative (làm gốc), và số lượng là absolute. Khi màn hình co giãn, biểu tượng số lượng vẫn bám sát giỏ hàng mà không bị chạy lung tung.

Ảnh hưởng deploy: Nếu lạm dụng absolute với đơn vị pixel cố định (px), khi đưa website lên hosting và xem trên điện thoại, giao diện sẽ bị vỡ. Thay vào đó, kết hợp relative và dùng đơn vị phần trăm (%) sẽ giúp website hiển thị tốt trên mọi thiết bị.

2. Quy trình đăng ký Domain, Hosting và vai trò của DNS

Quy trình: Đầu tiên, mua Tên miền (Domain - ví dụ: tenban.com). Sau đó, thuê Không gian lưu trữ (Hosting - nơi chứa code). Cuối cùng, cấu hình trỏ Domain về địa chỉ IP của Hosting.

Vai trò của DNS: DNS hoạt động như một "danh bạ điện thoại". Khi người dùng gõ tên miền, DNS sẽ dịch tên miền đó thành địa chỉ IP của máy chủ chứa website để trình duyệt biết đường tìm đến.

Công cụ Deploy miễn phí: Để deploy code tĩnh (HTML/CSS/JS) không cần mua server, ta có thể dùng GitHub Pages hoặc Netlify. Chỉ cần push code lên GitHub, hệ thống sẽ tự động build thành website.

Rủi ro bảo mật: Nếu cấu hình sai (không bật HTTPS, cấu hình sai phân quyền thư mục), tin tặc có thể đánh cắp dữ liệu người dùng khi truyền tải hoặc can thiệp vào mã nguồn website.

3. So sánh let, const và var trong JavaScript

So sánh: var có phạm vi hoạt động theo hàm (function scope) và bị hoisting (đưa lên đầu file nhưng không gán giá trị). let và const có phạm vi theo khối (block scope) như vòng lặp, if/else, giúp code an toàn hơn. const dùng cho hằng số không thay đổi.

Ví dụ tình huống lỗi thường gặp: Dùng var trong vòng lặp có hàm bất đồng bộ sẽ gây lỗi logic.

// Lỗi với var: Sẽ in ra 3 lần số 3
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100); 
}

// Khắc phục với let: Sẽ in ra 0, 1, 2
for (let j = 0; j < 3; j++) {
    setTimeout(() => console.log(j), 100); 
}

Cách Debug: Nhấn F12 mở Chrome DevTools, chọn tab Sources. Mở file JS, click vào số dòng để đặt Breakpoint (điểm dừng). Tải lại trang, code sẽ dừng tại đó, giúp ta kiểm tra giá trị của biến trong mục Scope để tìm ra lỗi.

4. Khái niệm Event Delegation trong JavaScript

Khái niệm: Thay vì gán sự kiện (event listener) cho từng phần tử con riêng lẻ, ta gán một sự kiện duy nhất cho phần tử cha chứa chúng. Kỹ thuật này dựa trên cơ chế "nổi bọt" (bubbling) của JavaScript.

Ví dụ ứng dụng: Trong một danh sách sản phẩm <ul id="productList">, khi bạn dùng JS thêm hoặc xóa các thẻ <li>, nếu gắn sự kiện trực tiếp vào thẻ li, các thẻ mới thêm sẽ không nhận được click. Nếu dùng Event Delegation gắn vào thẻ <ul>, mọi thẻ <li> mới sinh ra đều hoạt động bình thường.

Lợi ích tối ưu: Code ngắn gọn hơn, trình duyệt tiết kiệm được rất nhiều bộ nhớ vì chỉ phải quản lý 1 sự kiện thay vì hàng trăm sự kiện lẻ tẻ, giúp website chạy nhanh và mượt hơn.
